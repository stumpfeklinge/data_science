# Data Science Projects

В данной папке собраны учебные и практические проекты по **анализу данных и Data Science**.  
Каждый проект выполнен в **Jupyter Notebook** и демонстрирует полный аналитический цикл:

- загрузка и изучение данных  
- предобработка и очистка данных  
- исследовательский анализ (EDA)  
- визуализация данных  
- проверка гипотез  
- построение моделей машинного обучения  
- формирование аналитических и бизнес-выводов  

Основная цель — продемонстрировать навыки работы с реальными датасетами и решения аналитических задач.

# Проекты

## 1. Исследование надежности заемщиков

### Задача
Определить, влияет ли семейное положение и количество детей клиента на вероятность возврата кредита в срок.  
Результаты исследования могут быть использованы для построения системы **кредитного скоринга**.

### Данные

Исходные данные содержат **21 525 записей** со следующими характеристиками:

- `children` — количество детей  
- `days_employed` — трудовой стаж в днях  
- `dob_years` — возраст клиента  
- `education` — уровень образования  
- `family_status` — семейное положение  
- `income_type` — тип занятости  
- `total_income` — ежемесячный доход  
- `purpose` — цель кредита  
- `debt` — наличие задолженности по кредиту  

### Основные этапы анализа

- обработка пропусков  
- исправление аномальных значений  
- удаление дубликатов  
- приведение типов данных  
- категоризация доходов и целей кредита  
- анализ влияния факторов на возврат кредита  

### Ключевые выводы

**Значимые факторы:**

- уровень образования  
- семейное положение  
- цель кредита  

**Незначимые факторы:**

- количество детей  
- уровень дохода  

Кредиты на **недвижимость и свадьбу** возвращаются чаще, чем кредиты на автомобили и образование.

### Стек

Python, Pandas, NumPy

# 2. Сравнение музыкальных предпочтений Москвы и Санкт-Петербурга

### Задача

Проверить гипотезы о различиях в музыкальных предпочтениях пользователей сервиса **Яндекс.Музыка** из Москвы и Санкт-Петербурга.

### Данные

Датасет `yandex_music_project.csv` содержит:

- `user_id` — идентификатор пользователя  
- `track` — название трека  
- `artist` — исполнитель  
- `genre` — жанр музыки  
- `city` — город пользователя  
- `time` — время начала прослушивания  
- `day` — день недели  

### Проверяемые гипотезы

1. Активность пользователей зависит от дня недели по-разному в двух городах.
2. Музыкальные предпочтения различаются в начале и конце недели.
3. Москва и Санкт-Петербург предпочитают разные жанры.

### Основные результаты

- Активность пользователей действительно различается по дням недели.
- Музыкальные предпочтения городов **в целом очень похожи**.
- **Поп-музыка** — самый популярный жанр в обоих городах.

### Стек

Python, Pandas, Jupyter Notebook

# 3. Анализ рынка недвижимости Санкт-Петербурга

### Задача

Проанализировать архивные данные сервиса **Яндекс.Недвижимость**, чтобы выявить факторы, влияющие на стоимость квартир, и подготовить данные для системы автоматической оценки недвижимости.

### Данные

Датасет содержит **23 699 объявлений** о продаже квартир.

Основные параметры:

- цена квартиры  
- площадь  
- количество комнат  
- этаж  
- высота потолков  
- расстояние до центра  
- наличие парков и водоёмов  
- длительность продажи объявления  

### Методы анализа

- исследовательский анализ данных (EDA)  
- анализ распределений  
- обработка выбросов  
- корреляционный анализ  
- сегментационный анализ по локациям  

### Ключевые выводы

Основные факторы стоимости:

- **площадь квартиры**
- **удалённость от центра**
- **количество комнат**

Цены на жильё **существенно снижаются** по мере удаления от центра города.

### Стек

Python, Pandas, NumPy, Matplotlib, Seaborn

# 5. Анализ тарифов телеком-компании «Мегалайн»

### Задача

Сравнить тарифные планы **«Смарт»** и **«Ультра»** и определить:

- как пользователи потребляют услуги связи  
- различается ли средняя выручка по тарифам  
- влияет ли регион пользователя на выручку  

### Используемые данные

- `users` — информация о пользователях  
- `calls` — данные о звонках  
- `messages` — данные о сообщениях  
- `internet` — интернет-трафик  
- `tariffs` — описание тарифных планов  

### Основные этапы анализа

- предобработка данных  
- агрегация данных по пользователю и месяцу  
- расчёт месячной выручки  
- анализ потребления услуг  
- проверка статистических гипотез  

### Ключевые выводы

- средняя выручка тарифа **«Ультра» выше**, чем у «Смарт»  
- пользователи тарифа **«Смарт» чаще превышают лимиты**  
- регион проживания **не влияет** на размер выручки  

### Стек

# 6. Рекомендация тарифов мобильной связи

### Задача

Построить модель машинного обучения, которая будет рекомендовать пользователю подходящий тариф мобильной связи на основе его поведения.

Классификация тарифов:

- **0 — Smart**
- **1 — Ultra**

Минимальное требование к качеству модели:

**Accuracy ≥ 0.75**

---

### Данные

Используется датасет **`users_behavior.csv`**.

| Признак | Описание |
|---|---|
| calls | количество звонков |
| minutes | суммарное количество минут разговора |
| messages | количество SMS |
| mb_used | объём интернет-трафика |
| is_ultra | целевая переменная |

Размер датасета: **3214 наблюдений**.

---

### Используемые модели

- Decision Tree
- Random Forest
- Logistic Regression

Библиотеки:

- Pandas
- Scikit-learn

---

### Основные этапы работы

1. Загрузка и изучение данных  
2. Разделение данных на выборки:
   - train — 50%
   - validation — 25%
   - test — 25%
3. Обучение моделей машинного обучения  
4. Подбор гиперпараметров  
5. Оценка качества моделей  

---

### Результаты

Accuracy моделей на тестовой выборке:

| Модель | Accuracy |
|---|---|
| Decision Tree | 0.8047 |
| Random Forest | **0.8209** |
| Logistic Regression | 0.7550 |

Лучший результат показала модель **Random Forest**.

Дополнительные метрики:

- Precision: **0.8156**
- Recall: **0.4936**
- F1-score: **0.615**

---

### Итог

Модель **Random Forest** показала наилучшее качество классификации (**accuracy = 0.8209**), что превышает требуемый порог **0.75**.

Модель может использоваться для **рекомендации оптимального тарифного плана пользователям** на основе их поведения.

### Стек

Python, Pandas, Scikit-learn, Jupyter Notebook
